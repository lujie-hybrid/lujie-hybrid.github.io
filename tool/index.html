// jsconfig.json // 用于识别@sass这种别名，进行路径智能提示
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@img/*": ["src/assets/images/*"],
      "@comp/*": ["src/components/*"],
      "@u/*": ["src/utils/*"],
      "@a/*": ["src/api/*"]
    },
    "target": "ES6",
    "module": "commonjs",
    "allowSyntheticDefaultImports": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}

# koroFileHeader 的配置

打开文件 → 首选项 → 设置 → 搜索 fileheader → 在下面的 setting.json 里面随便点击进入一个 → 下面 json 里面的配置，进行替换和添加

{
  // 文件头部注释
  "fileheader.customMade": {
    "Date": "Do not edit",
    "FilePath": "Do not edit",
    "描述": "",
    "编辑者": "[your name]"
  },
  //函数注释
  "fileheader.cursorMode": {
    "Date": "Do not edit",
    "描述": "",
    "参数": "",
    "编辑者": "[your name]",
    "返回值": ""
  },
}

  • 使用方法：

    ctrl+alt+i 是头部注释，对应上面的 fileheader.customMade
    ctrl+alt+t 是函数注释，对应上面的 fileheader.cursorMode

# 代码段 的配置 vue.json

打开文件 → 首选项 → 用户片段 → 搜索 vue → 在下面的 vue.json 进入加入要编辑的代码片段。
代码片段去 该网站 (opens new window) 转化，需要的是转化后的 body 里面的。

// 左边是要转化的代码，如下面

<template>
  <div></div>
</template>

<script>
export default {
  components: {},
  data() {
    return {};
  },
  methods: {},
};
</script>

<style lang="scss" scoped></style>


转化后：

"": {
  "prefix": "",
  "body": [
    "<template>",
    "  <div></div>",
    "</template>",
    "",
    "<script>",
    "export default {",
    "  components: {},",
    "  data() {",
    "    return {};",
    "  },",
    "  methods: {},",
    "};",
    "</script>",
    "",
    "<style lang=\"scss\" scoped></style>"
  ],
  "description": ""
}

// vue.json 下面的指令就是lvue，会自动生成代码

  // same ids are connected.
  // Example:
