# 一级目录

目录路径 说明 层级
vue.config.js vue 的配置文件 1
package.json 包信息文件(依赖包和指令 scripts) 1
babel.config.js babel 的配置文件 1
.gitignore git 提交的忽略文件配置 1
.eslintignore eslint 忽略的配置 1
.eslintrc.js eslint 的配置文件 1
public 纯粹静态文件夹目录，此目录不走 webpack 资源分析 1
dist 打包文件夹,里面是打包后的文件 1
node_modules 依赖的文件，一般安装依赖后就会有 1
src 业务开发文件目录 1
mock mockjs 模拟数据目录 1
.env.development 开发环境的环境变量 1
.env.staging 测试环境的环境变量 1
.env.production 生产环境的环境变量 1

目录路径 说明 层级   
src/api 接口 api 目录 2
src/assets 静态资源目录(图片等)，这里面的会走 webpack 资源分析 2
src/components 组件目录，common 是通用组件目录，做了自动注册 2
src/components/index.js common 通用组件自动注册的 js
src/layout 布局目录，包括头部，左边导航和导航 tagView 2
src/plugins 项目用到的插件，如 elementui 2
src/router 路由定义的地方 2
src/store vuex 状态管理目录 2
src/utils 通用工具/方法 2
src/views 页面目录 2
src/styles 样式目录 2
src/App.vue 项目的主页面 2
src/main.js 项目的主入口文件，包含了众多配置的引入和使用 2
src/permission.js 项目的权限路由的控制文件 2
src/setting.js 一些配置的静态数据 2

# 功能点

  1. 别名 alias, 用于路径的 import 引入，比如 import xxx from "@img/logo.png"
configureWebpack: {
  name: name,
  resolve: {
    alias: {
      "@": resolve("src"),
      "@img": resolve("src/assets/images"),
      "@comp": resolve("src/components"),
      "@u": resolve("src/utils"),
      "@a": resolve("src/api"),
    },
  },
},
  1. @com/common 里面的通用组件的用法
// 比如 CommonTable.vue, 不用引入，直接使用，用Vue.component做了全局注册
使用： <v-common-table></v-common-table>
  1. api 自动注入

    @sass/api/* 已经对接口进行了自动注入，绑定到 Vue.prototype.$api 上
    使用方法如下：

// 文件名是驼峰写法
this.$api.文件名.方法名().then(()=>{...});

// 实例 (api/modules/user.js)
// this.$api.user.getList其实就是导出方法的引用
getData() {
  let { page, size } = this.pageInfo
  this.pageInfo.loading = true
  this.$api.user
    .getList({
      pageNo: page,
      pageSize: size,
      condition: this.searchCont,
      projectPhase: setAllCode(this.projectPhase),
      industryCode: setAllCode(this.industryCode),
      projectType: this.formVal.orderType,
      payStatus: this.formVal.payStatus,
    })
    .then((res) => {
      this.pageInfo.loading = false
      this.pageInfo.total = res.result.total
      this.tableData = res.result.records
    })
},

# 原理就是：api/*.js 里面每个 js 文件 export 导出的方法，整体绑定一个{}的 key 上，key 的名字就是文件名，然后将这个{}赋值给 Vue.prototype.$api
