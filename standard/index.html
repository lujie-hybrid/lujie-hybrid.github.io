# 前端规范

# 命名kebab-case or PascalBase

1️⃣ 除 components/** 文件夹命名

除了 components 文件夹下的子文件夹，统一使用小写字母开头的 kebab-case 风格

例子

  • /src/assets/404-images/
  • /src/views/user/base-info/

2️⃣ components 文件夹下的子文件夹

统一使用大写字母开头的 PascalBase 风格

组件类型：(通用组件 | 业务组件)

1️⃣  通用组件

  1. 功能性的组件，基于 element-ui 二次封装的组合性组件，比如 table+分页，表单根据配置自动生成select,input,checkbox,radio等，上传组件 等
  2. 基于第三方插件封装的 vue 文件，比如 echarts图表的封装等，这种会注册为全局组件，可以直接引用
  3. 基于第三方插件封装的 js 文件，这种一般可以定义成Class类，在index.js进行导出，比如 动画.js等，这种需要import引入
  4. 超过三次以上使用的业务组件，多个地方使用，为了方便，可以定义为通用组件，组件命名基于模块，见文知义

2️⃣  业务组件

  1. 页面专属组件，专属一个页面，这个一般是页面的结构，比如头部，底部，aside，内容区,弹框的划分，这个做的目的是防止页面过分庞大，杂乱，导致一个页面上千行代码
  2. 多个页面(<=3)有相同或相似布局，封装成组件，这种组件一般放在多个页面的上级模块的 components 目录下

说明：

  1. 通用组件放在 /src/components/ 下,比如 element-ui 的二次封装组件
  2. 页面专属的组件，放在各自页面下的 ./components 目录下，模块下多个（<=3）页面使用，放在当前模块目录下的 ./components 目录下， 超过3个地方 使用的组件，可以定义为通用组件
  3. 每个 components 目录下最多只有一层目录，且目录名称为组件的名称，统一使用大写字母开头PascalBase 风格，且目录下必须有 index.vue 或 index.js

例子

  • /src/components/Breadcrumb/index.vue
  • /src/components/animation/index.js
  • /src/views/layout/components/SideBar/index.vue

3️⃣ 除 *.vue 文件的 命名

除了 .vue 文件 ，统一使用小写字母开头的 kebab-case 风格

例子

  • render-dom.js / sign-up.css / index.html / company-logo.png
  • /src/views/msg/msg-list/msg-list-data.js

4️⃣ *.vue 的文件命名

除 index.vue 之外，统一使用大写字母开头的 PascalBase 风格

例子

  • /src/views/msg/msg-list/index.vue
  • /src/views/msg/msg-list/MsgListHeader.vue
  • /src/components/msg/msg-list/MsgWrap/MsgItem.vue

5️⃣ 命名要求：

命名注意点：

  • 名字的单词连接不要超过三级，project-manage-list 这种已经是极限
  • 命名严禁使用拼音与英文混合的方式，更不允许直接使用中文的方式
  • 杜绝完全不规范的缩写，避免望文不知义

错误示范 X

  • 文件夹/文件名：my-project-manage-list 四级 , zhijianhuo 拼音模式
  • abstract-class“缩写”命名成 abs-class；condition“缩写”命名成 condi

# Css/Sass 规范

如果不太了解 Sass，可以参考 官网 (opens new window)

提示  css/sass 的格式化可以使用 Vscode 的 Prettier 的快捷键 alt+shift+F

注意点：重要

1️⃣  项目可以选择使用 sass 或 css，通过 lang 进行区分，<style lang="scss" scoped></style>和<style scoped></style>
2️⃣ scoped 必须添加，这样就是局部 css，要不然就会成为全局样式，会污染环境，产生各种样式覆盖的 bug
3️⃣   加了 scoped 后，无法覆盖 elementui 默认样式，可参考 Scoped CSS (opens new window)

规范：

1️⃣  类名使用小写字母，以中划线分隔
2️⃣  名称总是使用可以反应元素目的和用途的名称，或其他通用的名称，代替表象和晦涩难懂的名称，简单来说就是 见名知义

/* 不推荐 */
.fw-800 {
  font-weight: 800;
}
/* 推荐 */
.heavy {
  font-weight: 800;
}

3️⃣ 禁止使用 ID 选择器及直接使用全局标签选择器，防止污染全局样式，推荐使用 class 选择器

/* 不推荐 */
#header {
  padding-bottom: 0px;
  margin: 0em;
}
p {
  line-height: 16px;
}
.header p {
  line-height: 16px;
}

/* 推荐 */
.header {
  padding-bottom: 0px;
  margin: 0em;
}
.header .header-item {
  line-height: 16px;
}

4️⃣  名称中划线-连接最多三级，不可超过 三级
5️⃣ 推荐使用 直接子选择器，而不是后代选择器，后代选择器可能会导致很多的样式覆盖的问题 \

/* 不是最完美的 */
.content .title {
  font-size: 2rem;
}
/* 推荐 */
.content > .title {
  font-size: 2rem;
}

6️⃣  尽量使用缩写属性

/* 不推荐 */
border-top-style: none;
font-family: palatino, georgia, serif;
font-size: 100%;
line-height: 1.6;
padding-bottom: 2em;
padding-left: 1em;
padding-right: 1em;
padding-top: 0;

/* 推荐 */
border-top: 0;
font: 100%/1.6 palatino, georgia, serif;
padding: 0 1em 2em;

7️⃣  省略 0 后面的单位

/* 不推荐 */
.header {
  padding-bottom: 0px;
  margin: 0em;
}

/* 推荐 */
.header {
  padding-bottom: 0;
  margin: 0;
}

8️⃣ 每个选择器及属性独占一行，这个是风格问题，可以使用 Prettier 配合 Vscode，一键格式化
9️⃣  sass 中的变量、函数、混合、placeholder 等 采用驼峰式命名 \

注意  覆盖 element-ui 样式时候，不要直接写 .el-input 这种全局的样式，这样会污染环境，需要父级加个 class，然后用后代选择器或直接子选择器进行连接，比如： 
/* 不推荐 */
.el-input {
  padding-bottom: 0px;
  margin: 0em;
}

/* 推荐 */
.header-form .el-input {
  padding-bottom: 0;
  margin: 0;
}
.header-form > .el-input {
  padding-bottom: 0;
  margin: 0;
}

# Javascript 规范

前言：

  1. Javascript 的代码检查和代码风格会使用 Eslint+Prettier 进行控制，只要不符合标准，都会有错误或者警告信息，可以配合 Vscode工具 可以实现 一键格式化
  2. 写代码时候，如果不太了解一些 Javascript 语言的注意点，可以参考 语言规范点

规范

1️⃣ 重要 优先使用 Es6+ 语法，不太了解 Es6 的可以参考 Es6 教程 (opens new window)

2️⃣  命名统一使用小写驼峰 lowerCamelCase 命名，尽量不要超过三个单词

<script>
export default {
  methods: {
    /**
     * @Date: 2022-02-23 16:02:34
     * @Desc:
     * @Return:
     * @Author: lujie
     * @param {Number} age
     * @param {Object} httpHeader
     */
    getHttpMessage(age = 18, httpHeader = {}) {
      const name = "张三";
      const userName = "李四";
    },
  },
};
</script>

3️⃣  method 方法命名推荐 动词+名词 形式

saveShopData / showShopInfo

4️⃣ "", and 0, and NaN, false,null,undefined 的判断

if(msg){
  ...   // 不是"", and 0, and NaN, false,null,undefined任何一个
}
if(!msg){
  ...  // 是"", and 0, and NaN, false,null,undefined 其中一个
}

5️⃣  判断 undefined

/* 不推荐 */
if (person === undefined) { ... }
/* 推荐 */
if (typeof person === 'undefined') { ... }

6️⃣ 注意 console.log，debugger 只能用于调试，使用完请尽快删除

7️⃣ 条件判断和循环最多三层，条件判断能使用 三目运算符 和 逻辑运算符 解决的，就不要使用条件判断，但是谨记不要写太长的三目运算符，如果超过 3 层请抽成函数

# Vue 规范

Vue 文件使用了 Eslint+Prettier 对代码错误和代码风格进行控制，只要不符合标准，都会有错误或者警告信息，可以配合 Vscode工具 可以实现 一键格式化

官网规范

1️⃣ 重要 必要性的 A级规则 详细参考  A 级 (opens new window) ，以下几个尤为重要

组件数据 (opens new window)
Prop 定义 (opens new window)
为 v-for 设置键值 (opens new window)
避免 v-if 和 v-for 用在一起 (opens new window)
为组件样式设置作用域 (opens new window)

2️⃣ 推荐 强烈推荐，增强可读性 B级规则 详细参考  B 级 (opens new window) ，以下几个重点关注

单文件组件文件的大小写 —— 我们使用的是单词大写开头 (PascalCase) (opens new window)
基础组件名 —— 我们是 E 开头，表示 efreight (opens new window)
模板中的组件名大小写 —— 我们使用的是所有的地方都使用 kebab-case (opens new window)
JS 中的组件名大小写 —— 特别是 import 和 name (opens new window)
Prop 名大小写 (opens new window)
模板中简单的表达式 (opens new window)
简单的计算属性 (opens new window)
指令缩写 —— 我们使用全部简写 (opens new window)

3️⃣ 警告 错误的使用方式，尽量避免 D级规则 详细参考  D 级 (opens new window)

额外补充

1️⃣ 重要 组件 Prop 的定义

  1. 必须使用 camelCase 驼峰命名
  2. 必须指定类型
  3. 必须加上注释，表明其含义
  4. 必须加上 required 或者 default，两者二选其一
  5. 如果有业务需要，必须加上 validator 验证
  6. 属性可以分离就分离，不要一直就用一个 Object 处理，这样不太好控制属性类型和默认值
  7. 当然一个组件属性也不要过分多，这个需要自己把控，属性个数控制在 9 个以内
props: {
  // 组件状态，用于控制组件的颜色
   status: {
     type: String, // 这个必须
     required: true,
     validator: function (value) {
       return [
         'success',
         'info',
         'error'
       ].indexOf(value) !== -1
     }
   },
   // 用户级别
   userLevel：{
      type: String,
      required: true
   },
   // 个数
   count: {
      type: Number,
      default: 0
   },
   // 传入的信息
   message: {
      type: Object,
      default: ()=>({name:'efreight'})
   }
}

2️⃣  vue 单文件标签顺序保持一致

<template></template>
<script></script>
<style></style>

3️⃣  script 标签内部结构顺序统一，没有用到就不用写

<script>
export default {
  props: {},
  data() {
    return {};
  },
  components: {},
  computed: {},
  watch: {},
  directives: {},
  filters: {},
  // 生命周期函数 created,mounted,...
  created() {},
  methods: {},
};
</script>

4️⃣  必须为 v-for 设置键值 key ,key 值使用唯一值，一般是 id ，尽量不要用 index ，特别是增删改的结构，用 index 会产生 bug
5️⃣ 注意  不要手动操作 DOM，优先考虑 ref
6️⃣  v-show 与 v-if 选择

如果运行时，需要非常频繁地切换，使用 v-show ；如果在运行时，条件很少改变，使用 v-if

# 注释规范 重要

规则：

1️⃣  头部注释，对该文件进行说明，组件/页面/文件

  • 通用功能性组件 /src/components/** 介绍该组件的功能 ，比如：通用的 table + 分页
  • 业务组件，介绍该组件 哪个模块/哪个页面/组件介绍
  • 页面，介绍该页面是 哪个模块/哪个页面
  • 文件，介绍该文件的 功能作用，干啥用的

2️⃣  methods 函数注释，使用区域注释 /*...*/ ，包括日期，描述，返回值，编辑者，参数

函数注释会使用 vscode 插件 koroFileHeader，规定模板统一快捷键 ctrl+win+t 注释

methods:{
  /**
   * @Date: 2022-02-23 16:02:34
   * @Desc: 获取信息
   * @Return: ....
   * @Author: lujie
   * @param {Number} age
   * @param {Object} httpHeader
   */
  getHttpMessage(age = 18, httpHeader = {}) {
    const name = "张三";
    const userName = "李四";
    return name + userName;
  },
}

3️⃣  变量和属性

每个变量都要注释，直接用 // 注释即可

4️⃣  凡是 import 引入的，不管是项目内组件，本地图片，还是第三方插件等都要用 // 注释做简单介绍
5️⃣   template 内部的注释，用 <!--注释内容--> 的模式分区块进行注释

<template>
  <!--头部导航-->
  <div></div>
  <!--中间内容区-->
  <div class="user-info"></div>
  <!--底部链接-->
  <div class="footer"></div>
</template>

6️⃣   样式的注释，用 /* 注释内容 */ 的模式进行注释

/* 头部导航 */
.header {
}
/* 中间内容区 */
.user-info {
}
/* 底部链接 */
.footer {
}

# Vue Router 规范

路由

1️⃣ path 里面单词复数统一使用小写字母开头的 kebab-case 风格

 {
   path: "/pm-manage",
   redirect: "/pm-manage/list",
   children:[
     {
       path: "list",
       name: "pm-manage_list"
     }
   ]
   ...
 }

2️⃣ name 的命名参照 path ，有 children 的情况下，要带上父级的 name，用 _ 隔开，name 除了path是 / 和 redirect都要填写

具体示例 可参考 路由
